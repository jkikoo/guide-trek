# Revue de complétude formateur — V2

Cette revue complète l’audit multi-domaines après tests d’usage du POC `index.html`.

## Verdict

Note avant V2 : **16 / 20**.

Note après V2 : **18 / 20**.

Le POC est plus solide parce qu’il couvre maintenant des situations formulées naturellement par un randonneur, sans lui demander de les traduire dans une catégorie experte.

## Corrections structurantes

### Ce qui manquait

- navigation : indices contradictoires, hors sentier involontaire, sentier barré ou impraticable ;
- bivouac : traces animales, nourriture, déchets et objets odorants ;
- secours : absence de réseau mobile, balise ou messagerie satellite ;
- matériel : éclairage perdu ou inutilisable ;
- santé : panique, anxiété forte ou jugement inhabituel ;
- blessure : ampoule ou douleur de pied qui modifie l’appui ;
- eau : eau stagnante, odorante, algues, carcasse ou contamination visible.

### Pourquoi c’était important

Un formateur ne juge pas seulement la réponse finale. Il juge aussi si les questions captent les signaux que les randonneurs savent observer sous stress.

Les cas ajoutés sont fréquents, ambigus et propices aux mauvaises décisions :

- continuer parce que le GPS affiche une trace malgré un terrain incohérent ;
- contourner un sentier barré hors trace ;
- dormir près de nourriture ou déchets non sécurisés ;
- chercher du réseau en dispersant le groupe ;
- compter sur le téléphone comme lampe principale ;
- banaliser une ampoule qui modifie l’appui ;
- traiter une eau douteuse comme une source normale ;
- prendre une décision engagée pendant une panique.

## Critères formateur appliqués

1. La question doit être observable sans exposer la règle de décision.
2. La recommandation doit rester faisable dans le contexte décrit.
3. La réponse ne doit pas donner une solution idéale quand il n’y en a pas.
4. Le retour, l’appel, l’attente, l’échappatoire et le bivouac doivent rester des options évaluées, pas des réflexes.
5. Une situation précoce doit être traitée avant la rupture complète.

## Résultat attendu

Le POC peut maintenant être utilisé comme modèle pour les prochains sujets à condition de conserver cette méthode :

- confronter les questions à des phrases naturelles de terrain ;
- rechercher des témoignages ou retours réels quand le domaine est nouveau ;
- auditer chaque domaine avec un profil vétéran formateur ;
- transformer chaque manque en scénario testable ;
- vérifier que le HTML autonome rend un avis sans exposer ses règles internes.
